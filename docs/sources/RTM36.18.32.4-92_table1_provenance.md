# Provenance: RTM 36.18.32.4-92 — Table 1 (<=1000V) → `kr_table`

## Что это

Таблица `kr_table` в SQLite заполнена значениями **Таблицы 1** РТМ **36.18.32.4-92**:
«Значения коэффициентов расчетной нагрузки Кр для питающих сетей напряжением до 1000 В».

Заполнение выполнено в `db/seed_kr_table.sql`.

## Размерность данных (контрольная)

- **unique_ne**: 35
- **unique_ki**: 9
- **total_rows**: 315 (матрица \(35 \times 9\))

Набор столбцов `Ki` (как в таблице): `0.10, 0.15, 0.20, 0.30, 0.40, 0.50, 0.60, 0.70, 0.80`.

## Унификация источника (`source`)

Для всех записей из Таблицы 1 используется единый идентификатор:

- `source = 'RTM36.18.32.4-92_tab1'`

Это обеспечивает однозначную атрибуцию происхождения строк `kr_table`.

## Как контролируется целостность

Файл `db/checks_kr_table.sql` выполняет DB-only проверки (должны проходить без ошибок):

- `COUNT(*) = 315`
- `COUNT(DISTINCT ne) = 35`
- `COUNT(DISTINCT ki) = 9`
- набор `ki` ровно равен ожидаемому (см. выше)
- `COUNT(DISTINCT source) = 1` и `source` равен `'RTM36.18.32.4-92_tab1'`

Примечание: проверки устроены так, что при несоответствии они завершаются ошибкой SQLite (деление на ноль).
