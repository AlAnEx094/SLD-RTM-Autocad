# TASK: CALC-ENGINEER (MVP-0.1)

Цель: реализовать расчётный “ядро” (без AutoCAD), строго соблюдая контракт `Kr`, и записывать результаты расчёта в SQLite.

## Deliverables

- `calc_core/kr_resolver.py`
  - `get_kr(db_path: str, ne: int, ki: float) -> float`
  - строгое поведение по `docs/contracts/KR_RESOLVER.md`
  - ошибки при:
    - отсутствии подходящего `ne_tab`
    - невозможности интерполяции (нет соседних `ki`)
- `calc_core/rtm_f636.py`
  - функции для расчёта строк и итогов щита по `docs/contracts/RTM_F636.md`
  - читает ввод из SQLite, пишет расчёт в SQLite (в отдельные таблицы)
  - GUID на сущностях
- `calc_core/__init__.py`
  - экспортирует основные функции/типы для использования инструментами

## Acceptance criteria

- `get_kr()` проходит тесты clamp/округления/интерполяции.
- Расчёт одного щита пишет результаты в БД и может быть экспортирован инструментами.

## Notes

- Ввод не модифицировать: исходные `ki`, `ne` и т.п. остаются как были; расчёт хранит производные поля отдельно.
- DWG не трогать (в будущем это отдельный слой).
